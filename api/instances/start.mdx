---
title: "Start Workflow Instance"
description: "Create and start a new workflow instance"
---

Create and immediately start a new workflow instance. This endpoint initializes the workflow, creates the first step instance, and begins execution. For workflows that start with a human task, the appropriate tasks will be created and assigned.

## Endpoint

```
POST /api/v1/workflow-instances
```

## Authentication

<Note>
  This endpoint requires authentication. Include a valid Bearer token in the Authorization header.
</Note>

## Request Body

<ParamField body="workflowDefinitionId" type="UUID">
  The unique identifier of the workflow definition to start. Either `workflowDefinitionId` or `workflowKey` must be provided.
</ParamField>

<ParamField body="workflowKey" type="string">
  The unique key of the workflow definition to start. Either `workflowDefinitionId` or `workflowKey` must be provided.
</ParamField>

<ParamField body="businessKey" type="string">
  Optional business identifier for this workflow instance (e.g., order number, ticket ID). If provided with the same workflow definition, the `runNumber` will increment automatically for retry scenarios.
</ParamField>

<ParamField body="inputData" type="object" default="{}">
  Initial input data for the workflow. This data is available throughout workflow execution and is immutable.
</ParamField>

<ParamField body="contextData" type="object" default="{}">
  Initial context data for the workflow. Context data can be modified as the workflow progresses through steps and is used to pass data between steps.
</ParamField>

<Note>
  Either `workflowDefinitionId` or `workflowKey` must be provided, but not both. The system will use the latest published version of the workflow definition, or the latest version if no published version exists.
</Note>

### Example Request

```bash
curl -X POST https://api.example.com/api/v1/workflow-instances \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "workflowKey": "expense-approval",
    "businessKey": "EXP-2024-001",
    "inputData": {
      "expenseAmount": 1500.00,
      "category": "travel",
      "description": "Flight to conference",
      "receipts": ["receipt_001.pdf"]
    },
    "contextData": {
      "expenseAmount": 1500.00,
      "category": "travel",
      "submittedBy": "user_123"
    }
  }'
```

## Workflow Execution Flow

When a workflow instance is started, the following occurs:

1. **Version Resolution**: The system selects the appropriate workflow version (latest published, or latest draft if none published)

2. **Instance Creation**: A new workflow instance record is created with status `running`

3. **Data Initialization**: Input and context data are stored in the workflow instance

4. **Action Recording**: A "start" action is recorded in the workflow history

5. **Status History**: Initial status change is logged (null → running)

6. **Step Execution**: The workflow enters the start step and begins executing:
   - **Start steps**: Immediately transition to the next step
   - **Human task steps**: Create and assign tasks to users/roles/groups, set workflow to `waiting`
   - **Subworkflow steps**: Launch child workflow, set parent to `paused`
   - **End steps**: Mark workflow as `completed` or `rejected`

7. **Notification**: For human tasks, notifications are created and sent to assigned users

8. **Response**: The fully initialized workflow instance is returned

## Error Responses

<ResponseField name="401 Unauthorized">
  Authentication token is missing or invalid.
  
  ```json
  {
    "detail": "Not authenticated"
  }
  ```
</ResponseField>

<ResponseField name="404 Not Found">
  Workflow definition with the specified ID or key does not exist.
  
  ```json
  {
    "detail": "Workflow definition not found."
  }
  ```
</ResponseField>

<ResponseField name="422 Unprocessable Entity">
  Request validation failed. Common causes:
  - Neither `workflowDefinitionId` nor `workflowKey` provided
  - Workflow version has no start step
  - Invalid data format
  
  ```json
  {
    "detail": "workflowDefinitionId or workflowKey is required."
  }
  ```
  
  ```json
  {
    "detail": "Workflow version has no start step."
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error">
  An unexpected server error occurred.
  
  ```json
  {
    "detail": "Internal server error"
  }
  ```
</ResponseField>

## Use Cases

- Start a new approval workflow for an expense report
- Initiate an onboarding workflow for a new employee
- Launch a document review process
- Trigger automated business processes
- Integrate workflows into external applications
- Retry failed workflows with a new run number
