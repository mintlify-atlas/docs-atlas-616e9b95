## Overview

Returns a list of all notifications for the current authenticated user, ordered by creation date (newest first). Notifications are created when workflow events occur, such as when tasks are assigned or workflows reach specific states.

## Endpoint

```
GET /api/v1/me/notifications
```

## Authentication

<Note>
This endpoint requires authentication. Include a valid authentication token in your request.
</Note>

## Request Parameters

This endpoint does not accept any query parameters or request body. It automatically returns notifications for the authenticated user.

## Response

Returns a list of notification objects ordered by creation date (newest first).

### Response Schema

<ResponseField name="items" type="array" required>
  Array of notification objects
  
  <ResponseField name="id" type="UUID" required>
    Unique identifier for the notification
  </ResponseField>
  
  <ResponseField name="workflowInstanceId" type="UUID">
    ID of the workflow instance this notification relates to
  </ResponseField>
  
  <ResponseField name="stepInstanceId" type="UUID">
    ID of the step instance this notification relates to
  </ResponseField>
  
  <ResponseField name="notificationType" type="string" required>
    Type of notification. Common types include:
    - `task_assigned` - A task has been assigned to the user
    - `workflow_completed` - A workflow has completed
    - `workflow_failed` - A workflow has failed
  </ResponseField>
  
  <ResponseField name="title" type="string" required>
    Notification title (e.g., "Action required: Review Document")
  </ResponseField>
  
  <ResponseField name="body" type="string" required>
    Notification message body providing details about the event
  </ResponseField>
  
  <ResponseField name="isRead" type="boolean" required>
    Whether the notification has been marked as read
  </ResponseField>
  
  <ResponseField name="readAt" type="string">
    ISO 8601 timestamp when the notification was marked as read (null if unread)
  </ResponseField>
  
  <ResponseField name="createdAt" type="string" required>
    ISO 8601 timestamp when the notification was created
  </ResponseField>
</ResponseField>

## Notification Types

<Note>
The `notificationType` field indicates why the notification was created:

- **task_assigned**: A human task has been assigned to you and requires action
- **workflow_completed**: A workflow you're involved with has completed
- **workflow_failed**: A workflow has failed or been rejected
</Note>

## Example Request

```bash cURL
curl -X GET https://api.example.com/api/v1/me/notifications \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN"
```

## Use Cases

- Display notifications in a user dashboard or inbox
- Build a notification center showing all user alerts
- Poll for new notifications to update UI in real-time
- Filter unread notifications to show pending items requiring attention
